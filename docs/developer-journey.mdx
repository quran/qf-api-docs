# Developer Journey

Start with [Request Access](/request-access), then choose the path that matches what you are building. These are alternatives, not steps to complete in order.

<DeveloperJourneyMap />

## How To Choose

- If users do not need to sign in, use the [Content APIs Quickstart](/docs/quickstart).
- If you want a production-shaped JavaScript app quickly, use [Starter With NPX](/docs/tutorials/oidc/starter-with-npx).
- If you are adding signed-in features to an existing app, use the [User APIs Quickstart](/docs/tutorials/oidc/user-apis-quickstart).
- If you need custom OAuth2, mobile, or token-flow details, use the [OAuth2 Tutorial](/docs/tutorials/oidc/getting-started-with-oauth2).
- If you already know the API family and need endpoint details, use the [API Reference](/docs/api-reference).
