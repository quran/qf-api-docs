## Get content resource snapshot



Fetch the current full snapshot for a syncable public content resource.

Clients use snapshot URLs from `RESOURCE_CREATE` and `RESOURCE_INVALIDATE` mutations to replace their local resource data.
Snapshots are current at fetch time and are not sequence-bound. Snapshot responses are not cacheable and return `Cache-Control: no-store`.
