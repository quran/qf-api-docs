## Get content resource snapshot



Use this endpoint to fetch a full replacement copy of one public content resource.

This is the companion endpoint for [GET /resources/sync](../resources-sync/): sync tells you that a full replacement is needed by returning `snapshot_url` on `RESOURCE_CREATE` and `RESOURCE_INVALIDATE` mutations.

Typical flow:
1. Bootstrap or poll with [GET /resources/sync](../resources-sync/).
2. If a mutation has a non-null `snapshot_url`, call this endpoint with that mutation's `resource_group` and `resource_id`.
3. Replace all local records for that resource with the returned `records`.
4. Continue applying later sync mutations normally.

Snapshots are current at fetch time and are not tied to a specific sync sequence. Snapshot responses are not cacheable and return `Cache-Control: no-store`.
