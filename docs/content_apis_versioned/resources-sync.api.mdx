## Sync public content resources



Bootstrap current public resources or poll ordered content mutations for syncable resources.

Use `bootstrap=true` with `resources` for the initial sync. The final bootstrap page returns `next_sync_token`.
Use `sync_token` with the same canonical `resources` filter for incremental sync.
`cursor` is used only for `next_page_url` pagination and is bound to the original filter and page size.

Sync responses are not cacheable and return `Cache-Control: no-store`.
